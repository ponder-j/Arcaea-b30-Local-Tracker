# 🌌 Arcaea Tracker - 个人专属的 B30 与本地成绩管理系统

**Arcaea Tracker** 是一款专为 Arcaea 核心玩家打造的现代化、纯净且高度自定义的网页端成绩管理工具。它不仅能帮助你精准记录每次突破的成绩，还能实时计算 B30、Max PTT，并生成精美的排位图片。

**已部署网站：https://arc-tracker.shop/**

## 🎯 核心特色功能

### 1. 📊 实时 B30 与 Max PTT 计算

输入单曲成绩后，系统会调用精确的 Arcaea PTT 计算公式，自动得出单曲 Potential。前端采用动态排序算法，无需刷新页面，Best 30 平均值、溢出区 (Overflow) 以及全局 Max PTT 实时呈现在你眼前。

### 2. 🔮 独创“目标预览”系统（硬核玩家最爱）

不知道接下来该推哪首歌收益最高？ 在每张成绩卡片的最右侧，带有 **“上 / 下 / 重置”** 三个微调按钮：

- **点按“上”箭头**：模拟你在该曲目中 **“少打出一个 Far”**（即增加 `10000000 / 总物量 / 2` 分）。
- **智能悬停锁定**：当你鼠标停留在卡片上进行模拟时，**卡片位置会被锁定**，方便你连续点击调整分数；当你移开鼠标，系统会立刻根据模拟的新 PTT 将这首歌**重新插入它应有的排名位置**！
- **全局联动**：模拟的分数会以显眼的**琥珀金色**高亮显示，并**实时计入你顶部的全局 Max PTT 中**。让你清晰地知道：“只要这首歌少 3 个 Far，我的总 PTT 就能上 13.00！”

### 3. 📝 智能谱面数据转换 (P / F / L 估算)

系统内置了曲目物量数据库，输入分数后，会自动为你推算出这把大概打了几个 Pure、几个 Far、几个 Lost，帮你更好地复盘。

### 4. 🗃️ 极速的数据导入与导出

- **批量导入**：支持一键上传 JSON 格式的成绩单，几秒钟即可完成几百首歌的录入。
- **精美图片生成**：内置 `html-to-image` 引擎，一键导出高清无水印的 B30 成绩图，排版优雅，完美适配暗色模式，非常适合在社交媒体上“晒卡”。

### 5. 👑 内置 Admin 管理面板

系统内置了 Admin 账号权限：

- 可以随时手动添加新曲目、分配各难度定数。
- 支持**无缝追加别名**（如搜“骨折光”、“fr”就能跳出 Fracture Ray），让查歌录入如丝般顺滑。

## 📖 快速上手指南

1. **注册与登录**：在首页输入你喜欢的用户名和密码直接注册即可进入系统。
2. **录入成绩**：
   - 在顶部的搜索框输入曲名（支持缩写/别名）。
   - 选择难度（PST/PRS/FTR/ETR/BYD），输入你的分数（如 `9951234`）。
   - 点击“保存成绩”，卡片将自动生成并飞入你的 B30 列表中。
3. **目标预览演练**：
   - 将鼠标移到某张成绩卡片上，注意最右侧弹出的控制面板。
   - 点击向上的箭头，看着分数上涨、单曲 PTT 提升。
   - 移开鼠标，看看这张卡片能冲到第几名，以及你的顶部 Max PTT 涨了多少！
4. **生成图片**：调整好你想展示的内容后，点击右上角的 **“生成 B30 图片”**，等待片刻即可下载高清大图。

## 🛠️ 技术架构

- **前端**：React + Vite + Tailwind CSS (极速响应，极致丝滑的 UI 体验)
- **后端**：Node.js + Express + MySQL
- **数据隔离**：支持多用户注册，每个用户的成绩数据互相隔离，互不干扰。

## 🚀 部署指南

### 1. 环境准备
- Node.js (建议 v16+)
- MySQL 数据库 (本项目使用 MySQL 存储数据)

### 2. 数据库配置
1. 在 MySQL 中创建数据库 `arcaea_tracker`。
2. 配置环境变量：在 `backend` 目录下创建 `.env` 文件，并根据 `server.js` 中的需求配置以下内容：
   ```env
   DB_HOST=localhost
   DB_USER=arcaea_user
   DB_PASS=你的数据库密码
   DB_NAME=arcaea_tracker
   JWT_SECRET=你自定义的任意密钥字符串
   PORT=3000
   ```
3. 运行根目录下的各环节 Python 数据导入脚本，将基础曲目数据和曲绘链接等写入数据库。（**执行前请务必确认脚本内 `"password": "YOUR_PASSWORD"` 处已修改为你实际的密码**）。

### 3. 后端启动
```bash
cd backend
npm install
node server.js
```
启动成功后，默认会监听在 `3000` 端口。

### 4. 前端启动与打包
开发模式调试：
```bash
# 在项目根目录执行
npm install
npm run dev
```

生产模式打包部署：
```bash
npm run build
```
打包完成后，`dist` 目录内的静态文件将会自动被后端的 `server.js` 托管（请检查 `app.use(express.static(path.join(__dirname, 'dist')));` 映射的正确性，你可以将前端打包出的 `dist` 复制/移动到 `backend/dist` 目录下）。
